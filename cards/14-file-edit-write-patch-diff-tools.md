Shared invariant:
- Mutating file tools must separate the model contract, runtime mutation, persisted/cache state, user diff display, debug/trace hooks, and eval evidence. Runtime mutation must be gated by a source-proven validation/permission/sandbox path, and display diffs must be derived from known before/after or committed-delta data rather than invented after the fact.

Claude-specific design:
- Claude exposes three source-specific model-visible mutation tools: `Edit` for exact string replacement, `Write` for full file creation/replacement, and `NotebookEdit` for cell-level notebook mutation (`src/tools/FileEditTool/types.ts:5-18`, `src/tools/FileWriteTool/FileWriteTool.ts:56-64`, `src/tools/NotebookEditTool/NotebookEditTool.ts:30-56`).
- Claude's stale-write guard is read-state based: existing-file edits/writes require a prior full/non-partial read, check file mtime after that read, and in some full-read cases compare content to avoid false positives (`src/tools/FileEditTool/FileEditTool.ts:275-310`, `src/tools/FileWriteTool/FileWriteTool.ts:198-221`).
- Claude's edit primitive is not a patch language. Patches are structured display/runtime output produced from exact replacement or full-content rewrite data (`src/tools/FileEditTool/utils.ts:234-350`, `src/tools/FileWriteTool/FileWriteTool.ts:359-379`).
