Shared invariant:
- A harness must not treat shell execution as just "run command and return stdout." Source-backed designs separate model-visible command contract, permission/sandbox gating, process launch, output transport, progress streaming, long-running/background state, interrupt/termination semantics, persisted output/process state, user display, and debug/eval traces.

Claude-specific design:
- Claude exposes a named `Bash` tool with a high-level shell command string and optional `run_in_background`; it does not expose an interactive stdin continuation tool for Bash in the cited source. Background continuation is represented as task notifications and output files (`src/tools/BashTool/BashTool.tsx:227-259`, `src/tools/BashTool/BashTool.tsx:555-623`, `src/tasks/LocalShellTask/LocalShellTask.tsx:105-171`).
- Claude's normal Bash output path writes stdout and stderr into one task-output file and polls tails for progress; foreground completion may inline output or persist large output separately for model consumption (`src/utils/Shell.ts:289-345`, `src/utils/task/TaskOutput.ts:21-31`, `src/tools/BashTool/BashTool.tsx:724-760`).
- Claude has multiple background triggers: explicit model `run_in_background`, timeout auto-background, assistant-mode auto-background, and user Ctrl+B backgrounding. These are source-specific and must not be generalized to Codex without matching source (`src/tools/BashTool/BashTool.tsx:965-1001`, `src/tools/BashTool/BashTool.tsx:1108-1125`, `src/tools/BashTool/UI.tsx:31-84`).
- Claude's `interrupt` abort reason is special for shell commands: the abort handler does not kill on that reason, allowing higher-level backgrounding/partial-output behavior; normal kill still uses process-tree kill (`src/utils/ShellCommand.ts:186-193`, `src/utils/ShellCommand.ts:337-347`).

Codex-specific design:
- Codex unified exec exposes a two-tool interactive protocol: `exec_command` opens or completes a command, and `write_stdin` writes/polls an existing session id. This is source-specific and should not be projected onto Claude Bash (`codex-rs/core/src/tools/handlers/shell_spec.rs:21-151`, `codex-rs/core/src/tools/handlers/unified_exec/exec_command.rs:122-327`, `codex-rs/core/src/tools/handlers/unified_exec/write_stdin.rs:69-102`).
- Codex keeps legacy `shell_command` as a separate mode/dispatch path. In unified-exec mode, `shell_command` remains registered dispatch-only while `exec_command/write_stdin` are the model-visible shell interface (`codex-rs/core/src/tools/spec_plan.rs:602-621`).
